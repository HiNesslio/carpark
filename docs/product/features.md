# 澳門停車場空位查詢系統

## Strategy (战略层)

**用戶畫像**：澳門駕駛者，每天面對「哪裡有車位？」的焦慮。他們可能在繁忙的街道上，需要快速、準確的停車場資訊來做出決策。現有方案要麼是分散的資訊，要麼是複雜的App，沒有一個簡潔直觀的解決方案。

**核心情感需求**：「一目了然的掌控感」——用戶希望在幾秒鐘內找到最近的可用停車場，消除四處尋找車位的焦慮。

**成功指標**：用戶平均停留時間 < 30秒（快速找到目標），每週回訪率 > 3次。

**核心調性**：快速、清晰、實時、可靠

## Scope (范围层)

### NOT (大膽砍掉)
- 不需要用戶註冊/登入系統 —— 增加使用門檻
- 不需要導航功能 —— 用戶已有Google Maps
- 不需要預約功能 —— 澳門停車場不支持
- 不需要歷史數據圖表 —— 增加複雜度，偏離核心需求

### MUST (必須要有)
1. **即時空位展示卡片** —— 停車場名稱、地址、總車位數、剩餘車位數、最後更新時間，用顏色直觀顯示空位狀態（綠=充足、黃=緊張、紅=已滿）
2. **自動刷新機制** —— 每60秒自動更新數據，用戶也可手動刷新
3. **排序功能** —— 按剩餘車位數排序，幫助用戶快速找到目標
4. **數據持久化** —— 後端定期獲取並保存API數據，減少對外部API的請求

## Structure (结构层)

### 頁面架構
單頁應用，垂直滾動設計。頂部是搜索和刷新區域，下方是停車場卡片網格。

### 注意力流動
1. **進入頁面** → 看到標題「澳門停車場即時空位」
2. **掃視概覽** → 卡片顏色快速傳達空位狀態
3. **篩選排序** → 點擊排序按鈕重新排列
4. **查看詳情** → 確認地址和更新時間

### 數據模型

| Entity | Fields |
|--------|--------|
| CarPark | id, name, address, totalSpaces, availableSpaces, latitude, longitude, updatedAt |

### API端點

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | /api/carparks | 獲取所有停車場數據 |
| POST | /api/carparks/refresh | 手動刷新數據 |

## Skeleton (框架层)

### 導航形式
無需導航欄。頂部固定標題欄 + 刷新按鈕，最大化內容區域。

### 關鍵頁面佈局
- **頂部區域**：標題 + 最後更新時間 + 刷新按鈕
- **排序控制**：排序下拉選單
- **主內容區**：停車場卡片網格（桌面3列，平板2列，手機1列）

### 簽名佈局模式
**交通燈式狀態卡片** —— 每張卡片左側有顏色條（綠/黃/紅），一眼識別空位狀態。

### 組件清單

| Component | Purpose | Pages |
|-----------|---------|-------|
| Header | 標題 + 刷新按鈕 + 更新時間 | Index |
| SortControl | 排序選擇器 | Index |
| CarParkCard | 停車場資訊卡片 | Index |
| StatusBadge | 空位狀態徽章 | CarParkCard |
| RefreshButton | 刷新按鈕（帶動畫） | Header |

## Surface (表现层)

### 色彩方案
交通燈配色方案，直觀傳達狀態：
- **主色**：深藍色 `oklch(0.45 0.15 250)` —— 專業、可信賴
- **成功色**：翠綠 `oklch(0.65 0.18 145)` —— 空位充足
- **警告色**：琥珀黃 `oklch(0.78 0.15 85)` —— 空位緊張
- **危險色**：珊瑚紅 `oklch(0.60 0.20 25)` —— 已滿

### 字體
- 標題：Inter —— 清晰現代
- 數字：Tabular-nums —— 數字對齊，方便比較

### 簽名視覺細節
卡片刷新時的「脈衝」動畫 —— 數據更新時卡片輕微閃爍，傳達「即時」感。
