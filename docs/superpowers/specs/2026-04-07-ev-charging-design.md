# 澳門公共停車場實時資訊 — EV 充電位功能設計

**日期**：2026-04-07
**狀態**：已批准

---

## 概述

在現有停車場應用中添加 EV 充電位顯示功能，並允許用戶過濾只顯示有 EV 充電位的停車場。

---

## 功能需求

1. **顯示 EV 充電位數量**：在每個停車場卡片上顯示該停車場的 EV 充電位數量
2. **EV 過濾模式**：用戶可以切換到「EV 模式」，只顯示有 EV 充電位的停車場
3. **數據來源**：從 CEM 網站 (`https://ev.cem-macau.com/zh/WhereToCharge`) 爬取數據
4. **數據緩存**：後端緩存 EV 數據 5 分鐘，減少請求次數

## 後端 API 設計

### `GET /api/carparks?lat={lat}&lng={lng}&ev_only={true|false}`

**請求參數**：
- `lat`：用戶緯度（必需）
- `lng`：用戶經度（必需）
- `ev_only`：可選，true/false，預設 false

**返回格式**：
```json
{
  "updated_at": "2026-04-07T10:30:00+08:00",
  "user_lat": 22.1987,
  "user_lng": 113.5439,
  "ev_mode": true,
  "carparks": [
    {
      "name": "栢力停車場",
      "distance_km": 0.3,
      "light_vehicle": {"available": 280, "total": 417},
      "heavy_vehicle": {"available": 0, "total": 0},
      "motorcycle": {"available": 30, "total": 32},
      "ev_charging": 4
    }
  ]
}
```

### 新增函數

1. **`fetch_ev_data()`**：從 CEM 網站爬取 EV 充電位數據
   - 使用 BeautifulSoup 解析 HTML
   - 提取停車場名稱和 EV 充電位數量
   - 緩存結果 5 分鐘

2. **`merge_ev_data(carparks, ev_data)`**：將 EV 數據合併到停車場列表
   - 使用模糊匹配（名稱包含關鍵字）對應停車場

## 前端設計

### 新增 UI 元素

1. **EV 切換開關**：
   - 位置：頂部標題欄右側
   - 樣式：滑動開關，開啟時顯示綠色
   - 文字：預設顯示「全部」，開啟後顯示「EV」

2. **EV 充電位標識**：
   - 位置：停車場卡片底部
   - 圖標：閃電 ⚡
   - 格式：如 `⚡ 4` 表示 4 個 EV 充電位

### 交互邏輯

- 預設顯示所有停車場（ev_only = false）
- 用戶點擊 EV 開關後，切換到 EV 模式
- EV 模式下只顯示有 EV 充電位的停車場
- 切換模式後立即刷新數據

## 數據映射策略

### 方案：預設映射表 + 網頁爬蟲補充

1. **預設映射**：為已知有 EV 充電位的停車場配置數據
2. **爬蟲補充**：從 CEM 網站獲取更多數據
3. **模糊匹配**：名稱包含關鍵字即視為匹配

## 錯誤處理

| 場景 | 處理方式 |
|------|----------|
| CEM 網站無法訪問 | 顯示「EV 數據暫時無法獲取」，不影響正常停車場顯示 |
| EV 數據為空 | 正常顯示停車場，隱藏 EV 標識 |
| 爬蟲超時 | 使用緩存數據，顯示「EV 數據可能過時」 |
